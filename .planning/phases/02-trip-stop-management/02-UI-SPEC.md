# Phase 2 — UI Design Contract

> Visual and interaction contract for Trip & Stop Management. Generated by gsd-ui-researcher from CONTEXT.md decisions and Phase 1 design patterns.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Tailwind CSS 4 only) |
| Preset | not applicable |
| Component library | none |
| Icon library | Lucide React or inline SVGs |
| Font | System default (inherit from browser) |

**Notes:**
- Phase 1 established Tailwind CSS 4 with Vite plugin — no shadcn initialization.
- All spacing and colors use Tailwind defaults (Slate, Blue, Red palettes).
- No component library — build from primitives (buttons, cards, inputs, modals) using Tailwind classes.

---

## Spacing Scale

Declared values (all multiples of 4 per Tailwind default scale):

| Token | Value | Tailwind Class | Usage |
|-------|-------|---|-------|
| xs | 4px | p-1, gap-1 | Icon gaps, inline padding |
| sm | 8px | p-2, gap-2, mb-2 | Compact element spacing, input padding |
| md | 16px | p-4, gap-4, mb-4 | Default element spacing, form field gaps |
| lg | 24px | p-6, gap-6, mb-6 | Section padding within cards |
| xl | 32px | p-8, gap-8, mb-8 | Trip card padding, dialog padding |
| 2xl | 48px | p-12, gap-12 | Section breaks, trip detail hero spacing |
| 3xl | 64px | p-16, gap-16 | Page-level margins |

Exceptions:
- **max-w-md:** Maximum width for modal dialogs (448px per Tailwind default).

**Phase 2 Specific Additions:**
- **Grid gaps:** gap-4 (16px) between trip cards, gap-3 (12px) between stop list items.
- **Split-panel gap:** gap-4 (16px) on trip detail page between stop list and map preview.
- **Mobile stacking:** Reduces gaps on small screens (gap-3 on mobile, gap-4 on tablet+).
- **Button padding:** py-2 (8px) for small buttons, py-4 (16px) for medium buttons — strict multiples of 4.

**Rationale:** All spacing uses multiples of 4px to maintain grid alignment. Button heights derived from padding multiples: py-2 = 8px + default font size 14px + line-height 1.5 ≈ 36px minimum (accessible). py-4 = 16px + font ≈ 44px (preferred touch target). No intermediate values (10px, 12px) used.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (text-sm) | 400 (regular) | 1.5 | Form inputs, stop descriptions, secondary text |
| Label | 14px (text-sm) | 600 (semibold) | 1.5 | Form field labels, button text, small headings |
| Heading | 20px (text-xl) | 600 (semibold) | 1.2 | Card titles, section heads, stop name in list |
| Display | 28px (text-3xl) | 700 (bold) | 1.1 | Page title (trip name in hero), main heading on empty state |

**Implementation Notes:**
- **14px (text-sm):** `text-sm` class used in Phase 1 — maintain for consistency.
- **20px (text-xl):** `text-xl` + `font-semibold` for trip card titles.
- **28px (text-3xl):** `text-3xl` + `font-bold` for trip hero title (established in Phase 1 Signup).
- **Font weights:** Use 2 primary weights only: `font-semibold` (600) for labels/headings and hierarchy, `font-bold` (700) for display/main headings. Body uses `font-normal` (400).
- **Line height:** Tailwind defaults: 1.5 for body (normal), 1.2 for headings (tight).

**Phase 2 Specific Additions:**
- **Stop number badge:** 12px (text-xs) + `font-semibold` for "Stop 1", "Stop 2", etc. (hierarchy via size, not weight increase).
- **Empty state heading:** 28px (text-3xl) + `font-bold` — "Plan your first road trip".
- **Empty state body:** 14px (text-sm) + `text-gray-600` — supporting copy.
- **Card badge (stop count):** 12px (text-xs) + `font-semibold` + gray background.

**Weight Consolidation:** Removed intermediate weight 500 (medium). Labels and headings now use 600 (semibold) for consistency. Display tier uses 700 (bold). This reduces font-weight variance and simplifies CSS.

**Shadow Hierarchy:**
- **No shadow:** Flat surfaces (input fields, dialog background).
- **Shadow-md:** `shadow-md` — small cards, hover state on buttons.
- **Shadow-lg:** `shadow-lg` — trip cards (visual weight, clickable).
- **Shadow-xl:** Not used in Phase 2.
- **Shadow-2xl:** `shadow-2xl` — modal dialogs, hero section overlay.

**Color Transitions:**
- **Button hover:** `hover:bg-blue-700` (darker blue), `transition-colors` for smooth effect.
- **Disabled state:** `disabled:bg-blue-400` (muted blue), `disabled:cursor-not-allowed`.
- **Focus ring:** `focus:ring-2 focus:ring-blue-500` (2px outline, blue accent).

## Focal Points (Visual Hierarchy)

**Dashboard Page:**
- **Primary focal point:** Trip card grid with hero cover photos (high visual weight via shadow-lg, large cards, cover imagery).
- **Secondary focal point:** Empty state ("Plan your first road trip") when no trips exist.
- **Tertiary focal point:** "+ New Trip" button (blue accent, persistent at top).

**Trip Detail Page:**
- **Primary focal point:** Hero banner at top with trip name overlay, cover photo background (shadow-2xl depth, large text size 28px).
- **Secondary focal point:** Stop list on left (40% width tablet+) — numbered list with editable items.
- **Tertiary focal point:** Map preview on right (60% width tablet+) — shows route and markers.

| **Stop List Heading** | "Stops ({N})" | Section heading in trip detail page |

| **Stop Address Label** | "Address" | Form field label when adding/editing stop |
| **Stop Description Label** | "Description" | Form field label (optional field) |
| **Stop Start Date Label** | "Start date" | Form field label |
| **Stop End Date Label** | "End date" | Form field label |
| **Drag Handle Tooltip** | "Drag to reorder" | Aria-label for grip icon (accessibility) |
| **Edit Stop Button** | "Edit" | Button text (verb) |
| **Delete Stop Button** | "Delete" | Button text (verb) |
| **Delete Stop Confirmation** | "Delete this stop?" | Dialog heading |
| **Delete Stop Confirm Copy** | "This action cannot be undone. Are you sure?" | Dialog body text |
| **Delete Stop Confirm CTA** | "Delete" | Red destructive button in confirmation dialog |
| **Delete Trip Confirmation** | "Delete this trip?" | Dialog heading |
| **Delete Trip Confirm Copy** | "This action cannot be undone. All {N} stops will be deleted." | Dialog body — contextual with stop count |
| **Delete Trip Confirm CTA** | "Delete" | Red destructive button |
| **Edit Trip Button** | "Edit" | Button on trip card or header |
| **Edit Trip Modal Title** | "Edit trip" | Modal heading for trip edit |
| **Loading Skeleton Text** | "(Skeleton card)" | Placeholder while fetching trip/stop data |
| **Error Toast (Trip Create)** | "Failed to create trip. {error detail}" | Toast notification on API error |
| **Error Toast (Stop Add)** | "Failed to add stop. Please check the address and try again." | Toast for address validation/API error |
| **Error Toast (Delete)** | "Failed to delete {trip/stop}. Please try again." | Fallback error copy for delete operations |
| **No Results (Address Search)** | "No addresses found. Try a different search." | Shown when Nominatim returns 0 results |
| **Rate Limit (Unsplash)** | "Photo unavailable. Using placeholder." | Toast/note when Unsplash 50/hr limit hit |

**Destructive Actions (all require confirmation):**
1. **Delete Trip** — Confirmation dialog with red button. Copy: "Delete this trip and all {N} stops?"
2. **Delete Stop** — Confirmation dialog with red button. Copy: "Delete this stop?"

**Pattern Notes:**
- All primary CTAs use specific verb + noun (not "Submit", "OK", "Yes").
- Destructive actions always say "Delete" (not "Remove" or "Cancel").
- Error messages always include context and suggested next step.
- Empty state copy is action-oriented (what to do, not just what's missing).

| State | Visual | Implementation |

## Loading & Error States

### Loading (Skeleton Screens)
- **Trip Card Skeleton:** Gray placeholder card (same size as trip card) with animated shimmer.
- **Stop List Skeleton:** 3 gray bars (16px tall each) stacked with gap-3.
- **Map Skeleton:** Gray rectangle (200px tall) with animated shimmer.

### Error States (Toast Notifications)
| Scenario | Copy | Duration | Color |
|----------|------|----------|-------|
| Trip create failed | "Failed to create trip. {API error detail}" | 5 seconds | Red background (`bg-red-50`), red text |

## Accessibility Notes

- **Semantic HTML:** Use `<button>`, `<form>`, `<label>`, `<input type="date">`.
- **ARIA Labels:** Drag handle: `aria-label="Drag to reorder"`. Destructive buttons: `aria-label="Delete this trip (cannot be undone)"`.
- **Focus Management:** Modal dialogs trap focus (cycle through focusable elements). Close on Escape key.
- **Color Contrast:** All text meets WCAG AA (4.5:1 for normal text, 3:1 for large text).
- **Touch Targets:** Minimum 44px × 44px for mobile (buttons, drag handle).
