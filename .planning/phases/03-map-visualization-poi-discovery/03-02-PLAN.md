3. Create `frontend/src/hooks/useRoute.js`:
   - Import { useState, useEffect, useCallback } from 'react'
   - Import { getRoute } from '../services/routing.api'
   - Export `function useRoute(tripId, stops)`:
     - State: `route` (null), `isLoading` (false), `error` (null)
     - useEffect that triggers when `tripId` or `stops` change:
       - Count stops with valid coords: `stops.filter(s => s.address_lat && s.address_lon).length`
       - If fewer than 2, set route to null and return (no fetch)
       - Set isLoading true, error null
       - Call getRoute(tripId)
       - On success: set route to response.route
       - On error: set error to err.response?.data?.error || 'Route unavailable'
       - Finally: set isLoading false
     - Dependency array: `[tripId, stops.length, stops.map(s => s.id + s.address_lat + s.address_lon).join(',')]`
       - This ensures re-fetch on stop add/remove/reorder/address-change but NOT on description edits
     - Export `refetchRoute` callback that re-runs the fetch
     - Return { route, isLoading, error, refetchRoute }
